毎回定義すること

以下のルールで開発してください。

・既存機能を壊さない
・変更前に実装内容を説明する
・ファイルを削除しない
・エラーが出たら原因を説明する
・1回の作業は1機能だけにする

今回の変更内容をまとめてください。

・追加したファイル
・変更したファイル
・作成したGameObject
・Unity上で手動設定が必要なもの

を一覧で教えてください。

何か作業した後にすること

Phase〇（Hex Grid System）を実装してください。

コンパイルエラーがないことを確認
必要なファイルのみ変更
実装後に git status を確認
git commit -m "Phase1: Hex Grid System" を実行
git push まで行ってください

こうすると、実装からGitHubへの反映まで一連の流れで進めやすくなります。

初回設定が終わったので、次回以降は git push だけでアップロードできます。
